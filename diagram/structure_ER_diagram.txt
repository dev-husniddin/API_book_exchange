 
 Структура  ER-диаграммы и связи между сущностями в проекте "Book Exchange":

Сущности:
Книга (Book): Сущность, которая представляет информацию о книге. У нее есть атрибуты, такие как id (уникальный идентификатор книги), title (название книги), author (автор книги), publisher (издательство), publication_year (год издания), description (описание книги), cover_image (изображение обложки), selling_price (цена продажи), rental_price (цена аренды), format (формат книги), availability_status (статус доступности), isbn (международный стандартный номер книги), rating (рейтинг книги), page_count (количество страниц) и language (язык книги).

Пользователь (User): Сущность, которая хранит информацию о зарегистрированных пользователях платформы. У нее есть атрибуты, такие как id (уникальный идентификатор пользователя), username (имя пользователя), email (электронная почта пользователя), password (пароль), first_name (имя), last_name (фамилия), profile_image (изображение профиля), bio (информация о пользователе), preferences (предпочтения пользователя), registration_date (дата регистрации) и activation_status (статус активации).

Категория (Category): Сущность, представляющая категории, к которым может принадлежать книга. У нее есть атрибуты, такие как id (уникальный идентификатор категории), name (название категории) и description (описание категории).

Автор (Author): Сущность, содержащая информацию об авторах книг. У нее есть атрибуты, такие как id (уникальный идентификатор автора), name (имя автора) и biography (биография автора).

Заказ (Order): Сущность, представляющая заказы пользователей. У нее есть атрибуты, такие как id (уникальный идентификатор заказа), book_id (идентификатор книги, на которую делается заказ), user_id (идентификатор пользователя, сделавшего заказ), order_type (тип заказа - покупка, аренда или обмен), order_date (дата заказа), order_status (статус заказа), quantity (количество книг в заказе) и delivery_address_id (идентификатор адреса доставки).

Обмен (Exchange): Сущность, которая представляет информацию о процессе обмена книг между пользователями. У нее есть атрибуты, такие как id (уникальный идентификатор обмена), book_offered_id (идентификатор книги, которую предлагают для обмена), book_requested_id (идентификатор книги, которую хотят получить в обмен), user_offering_id (идентификатор пользователя, предлагающего обмен), user_requesting_id (идентификатор пользователя, желающего получить книгу в обмен), exchange_status (статус обмена), start_date (дата начала обмена) и end_date (дата окончания обмена).

Избранное (Favorites): Сущность, которая позволяет пользователям добавлять книги в свой список избранного. У нее есть атрибуты, такие как id (уникальный идентификатор), user_id (идентификатор пользователя, добавляющего в избранное) и book_id (идентификатор книги, добавляемой в избранное).

Аудиокнига (Audiobook): Сущность, которая хранит информацию об аудиокнигах. У нее есть атрибуты, такие как id (уникальный идентификатор аудиокниги), book_id (идентификатор книги, которая соответствует аудиокниге), audio_file (файл с аудиокнигой) и duration (продолжительность аудиокниги).

Электронная книга (Ebook): Сущность, которая хранит информацию об электронных книгах. У нее есть атрибуты, такие как id (уникальный идентификатор электронной книги), book_id (идентификатор книги, которая соответствует электронной книге) и file (файл с электронной книгой).

Отзыв о пользователе (UserReview): Сущность, которая хранит отзывы пользователей о других пользователях. У нее есть атрибуты, такие как id (уникальный идентификатор отзыва), user_reviewed_id (идентификатор пользователя, о котором написан отзыв), user_reviewer_id (идентификатор пользователя, который написал отзыв), rating (оценка) и comment (комментарий).

Адрес доставки (ShippingAddress): Сущность, представляющая адрес доставки для заказов. У нее есть атрибуты, такие как id (уникальный идентификатор адреса доставки), user_id (идентификатор пользователя, которому принадлежит адрес), address_line1 (первая строка адреса), address_line2 (вторая строка адреса, дополнительная информация), city (город), state (штат), country (страна) и postal_code (почтовый индекс).
Связи:
Каждая книга может принадлежать одной или нескольким категориям (многие-ко-многим).
Это означает, что у каждой книги может быть много категорий, и каждая категория может содержать много книг. Для установления такой связи необходима дополнительная таблица, которая будет хранить связи между книгами и категориями.

Каждая книга может иметь одного или нескольких авторов (многие-ко-многим).
Подобно связи с категориями, для установления связи "многие-ко-многим" между книгами и авторами также требуется дополнительная таблица.

Пользователи могут добавлять книги в свой список избранного (многие-ко-многим).
Также здесь используется дополнительная таблица для установления связи "многие-ко-многим" между пользователями и книгами, которые они добавили в избранное.

Каждый заказ относится к одной книге и одному пользователю (многие-к-одному).
Заказ связан с одной книгой, на которую делается заказ, и с одним пользователем, который сделал заказ.

Каждый обмен связан с двумя книгами и двумя пользователями - предлагающим обмен и желающим получить книгу (многие-к-одному).
Обмен связан с двумя книгами - той, которую предлагают в обмен, и той, которую хотят получить в обмен. Также связаны два пользователя - предлагающий обмен и желающий получить книгу.

У каждого обмена есть две стороны - предлагаемая книга и желаемая книга (многие-к-одному).
Обмен связан с двумя книгами - одной, которую предлагают в обмен, и другой, которую хотят получить в обмен.

Каждый отзыв о пользователе связан с двумя пользователями - тем, о котором отзыв, и тем, кто оставил отзыв (многие-к-одному).
Отзыв связан с пользователем, о котором написан отзыв, и с пользователем, который оставил отзыв.

Каждый заказ может быть доставлен по одному адресу (многие-к-одному).
Каждый заказ связан с одним адресом доставки.

Каждая аудиокнига и электронная книга связаны с одной книгой (один-к-одному).
Каждая аудиокнига и электронная книга связаны с одной книгой, к которой относятся, используя уникальный идентификатор книги (book_id).
